Chapter 1: Internship

1. Server Security

1.1 Context

During my internship at the Campus Provence Verte IT department, one of my first major missions involved securing all the establishment's main servers. These servers are the nerve center of the campus's digital infrastructure: they handle critical functions such as centralized user authentication, hosting essential services (databases, DNS, DHCP, Samba, etc.), client computer management, and inter-system communication coordination within the local network.

Their availability, reliability, and security are therefore paramount to ensuring the continuity of the establishment's educational and administrative activities. A failure or compromise of the servers could impact many users, such as teachers, students, or administrative staff, and compromise data confidentiality, integrity, or availability.

Main Server Infrastructure
  • SE4AD - Main domain controller (Debian 12 Bookworm)
  • SE4FS - File server and network services (Debian 12 Bookworm)
  • AdGuard DNS - DNS Filter (Debian 11 Bullseye)
  • Charlemagne - Windows Server for administrative documents

All servers (except Charlemagne) are hosted on a virtualization infrastructure based on Proxmox VE, installed on a physical machine equipped with SSD drives in RAID 1. This solution allows flexible, centralized, and secure management of the various virtual machines.

1.2 Tools and Technologies Used

Debian Linux

Migration from Debian 11 Bullseye to Debian 12 Bookworm to improve security and benefit from the latest updates.

Proxmox VE

Virtualization platform allowing centralized management of virtual machines, creation of backup clones, and resource reallocation.

SambaEdu

Centralized management solution for users, groups, and client computers. Integration with Active Directory for authentication and security policies.

Security Services
  • UFW (Uncomplicated Firewall) - Simplified firewall for Linux
  • Fail2ban - Protection against brute-force attacks
  • SSH with key authentication - Secure server access
  • FreeRADIUS - Network authentication server
Network Services
  • Apache2 - Web server
  • MariaDB - Database
  • DNS/DHCP - Essential network services
  • NFS/TFTP - File transfer protocols

1.3 Tasks Performed

System Updates

Migration from Debian 11 to 12 on SE4AD and SE4FS to benefit from security patches.

Firewall Configuration

Activation and setup of UFW on all Linux servers with specific rules (SSH, HTTP, HTTPS, DNS, etc.).

SSH Security

Setup of SSH key authentication, disabling password authentication, and changing the default port.

Fail2ban Installation

Configured to protect services against brute force, with custom rules for SSH, Apache, etc.

Backup and Cloning

Systematic creation of full clones of each VM before any sensitive modification. (Pre-modification clones, tests on cloned environments, pre-production validation).

1.4 Problems Encountered and Solutions

Samba connection block

After activating UFW on SE4AD and SE4FS, user accounts could no longer log into Windows workstations.

Solution: UFW logs analysis, identification of missing TCP 53 and UDP 389 ports, adding specific firewall rules.

APT update issues

Errors blocked server updates due to invalid repositories in /etc/apt/sources.list.

Solution: Commenting out defective lines in APT sources files, followed by restarting updates.

Unstable SSH connection

SE4AD listened on a custom SSH port (6602), disrupting certain automated operations requiring standard SSH.

Solution: Creation of a `~/.ssh/config` file to manage the non-standard port, configuring a cron job to maintain the SSH key, then creating a script to temporarily switch to port 22 during updates.

Secure update automation

SE4FS update failed when SSH connection to SE4AD was required, as port 22 was not open.

Solution: Creation of a `secure_upgrade.sh` script that temporarily switches SE4AD to port 22, dynamically opens the port in UFW, executes the update, and restores the initial configuration.

2. Centralized Computer and User Management

2.1 Context

The second major mission of my internship was to optimize and strengthen the centralized management of IT workstations and users within the Campus Provence Verte. This mission follows the securing of servers, as it aims to improve operational efficiency while maintaining a high level of security.

The campus has a substantial IT park composed of several classrooms equipped with computers, teacher workstations, and administrative workstations. The management of this entire setup is a constant challenge for the IT team, particularly concerning software deployment, security updates, user profile management, and preventive maintenance.

2.2 Tools and technologies used

Active Directory

Microsoft directory service integrated with SambaEdu for centralized management of users, groups, and security policies across the network.

Group Policy

Group Policy management system allowing centralized configuration and control of user and computer environments.

LDAP / Kerberos

Authentication and authorization protocols used to secure communications between client computers and the domain controller.

2.3 Tasks Performed

Group Policy Optimization

Review and optimization of Group Policy Objects (GPOs) to improve login times and standardize user environments.

User Profile Management

Setup of optimized roaming profiles and folder redirection to improve the user experience.

Deployment Automation

Creation of automated deployment scripts for software and updates, significantly reducing manual workload.

Monitoring and Maintenance

Setup of monitoring tools to track workstation performance and proactively detect issues.

2.4 Problems Encountered and Solutions

GPO Incompatibility with Windows 11 Pro

Certain group policies did not apply to Windows 11 Pro machines because they were designed for Enterprise or Education editions.

Solution: Manual verification of applied GPOs, deletion or adaptation of incompatible policies, and replacement with functional alternatives on the Pro edition.

Secure channel break with domain

After a Samba domain migration, several client computers lost their secure channel, preventing authentication and GPO application.

Solution: Creation of a `.bat` script executing a PowerShell command to reset the secure channel. Manual deployment via USB key. Network verification (connections, DNS) in case of failure.

Lack of server monitoring

The absence of monitoring tools made it difficult to detect anomalies on critical servers in real time.

Solution: Installation of Glances (CLI) for synthetic tracking, and Netdata (web interface) for detailed graphical monitoring with alerts and history.

Network congestion between Proxmox VMs

Several VMs shared the same physical network interface, leading to slowdowns, GPO application delays, and occasional disconnections.

Solution: Creation of distinct network bridges on the Proxmox server, assigning a dedicated physical interface to each critical VM (SambaEdu, backup, supervision), immediately reducing latency.

Overall Internship Summary

These two main missions significantly strengthened the security and efficiency of the IT infrastructure at Campus Provence Verte. Securing the servers established a solid foundation for the entire information system, while optimizing centralized management improved the user experience and reduced the administrative workload.

Technical Skills Acquired

  • Advanced Linux system administration
  • Network infrastructure security
  • Active Directory and LDAP management
  • Virtualization with Proxmox VE
  • Scripting and automation

Measurable Impact

  • Complete security of critical servers
  • 60% reduction in login times
  • 80% automation of administrative tasks
  • Improvement in system stability
  • Team training on new tools
